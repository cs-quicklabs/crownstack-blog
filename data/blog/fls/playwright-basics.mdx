**## Summary**

This document outlines the objectives of the Fast Learning Sprint, #FLS-09, focused on Playwright Basics. Upon completion, participants should be proficient in using Playwright for basic web automation and testing in their projects.

**## Who Can Participate**

Any member of the Development or QA team who is committed to learning Playwright basics and can dedicate time to demonstrate their acquired skills at the end of the FLS. Participants should be able to balance this learning with ongoing client project responsibilities.

**## Curriculum**

The following beginner-friendly topics will be covered, designed to be achievable within 15 days:

**### Introduction to Playwright**

- \***_Overview_**: Understand what Playwright is and its use cases for web automation and testing.
- \***_Playwright Architecture_**: Learn how Playwright interacts with browsers.
- \***_Installation and Setup_**: Set up Playwright in a project environment.
- \***_Playwright CLI and Configuration_**: Use the Playwright CLI and configure basic settings.
- \***_Reference_**: [Official Playwright Documentation - Getting Started](<[https://playwright.dev/docs/intro](https://playwright.dev/docs/intro)>)

**### First Steps with Playwright**

- \***_Write Your First Test_**: Create a simple Playwright test script.
- \***_Run Tests Across Browsers_**: Execute tests on Chromium, Firefox, and WebKit.
- \***_Debugging Tests_**: Use Playwright's debugging tools (e.g., inspector, trace viewer).
- \***_Reference_**: [YouTube - Playwright Tutorial for Beginners](<[https://www.youtube.com/watch?v=2f2bL6kFctY](https://www.youtube.com/watch?v=wuWLpsRwB5o&list=PLhW3qG5bs-L9sJKoT1LC5grGT77sfW0Z8&index=4)>)

**### Core Concepts**

- \***_Selectors and Locators_**: Use Playwright locators to interact with web elements.
- \***_Actions_**: Perform actions like click, type, and navigate.
- \***_Assertions_**: Validate test outcomes using Playwright's assertion methods.
- \***_Handling Dynamic Content_**: Manage dynamic web elements (e.g., waiting for elements).
- \***_Reference_**: [Playwright Documentation - Locators](<[https://playwright.dev/docs/locators](https://playwright.dev/docs/locators)>)

**### Test Organization**

- \***_Writing Modular Tests_**: Structure tests for reusability and clarity.
- \***_Grouping Tests with Describe Blocks_**: Organize tests using `describe` blocks.
- \***_Setup and Teardown_**: Use before/after hooks for test setup and cleanup.
- \***_Reference_**: [Playwright Documentation - Test Organization](<[https://playwright.dev/docs/best-practices](https://playwright.dev/docs/best-practices)>)

**### Playwright Test Runner**

- \***_Configuring the Test Runner_**: Set up basic test runner options.
- \***_Running Tests in Parallel_**: Execute tests concurrently for efficiency.
- \***_Generating Test Reports_**: Generate and interpret basic test reports.
- \***_Reference_**: [Playwright Documentation - Test Runner](https://playwright.dev/docs/test-running)

**### Page Object Model (POM)**

- \***_Creating Reusable Page Objects_**: Build page objects to encapsulate page interactions.
- \***_Organizing Tests with POM_**: Apply POM to structure tests effectively.
- \***_Reference_**: [Article - Playwright Page Object Model](<[https://www.lambdatest.com/learning-hub/playwright-page-object-model](https://www.lambdatest.com/learning-hub/playwright-page-object-model)>)

**### API Testing**

- \***_Making API Requests_**: Use Playwright to send API requests.
- \***_Validating API Responses_**: Verify API response data and status.
- \***_Reference_**: [Playwright Documentation - API Testing](<[https://playwright.dev/docs/api-testing](https://playwright.dev/docs/api-testing)>)

**## How to Participate**

1. Join the #fls Slack channel to commit to this learning sprint.

2. Only join if you can complete the curriculum within the 15-day period while managing client work.

3. We seek serious participants, not spectators.

**## Expectations from Candidates**

At the end of the FLS, participants will engage in a group discussion to demonstrate their Playwright knowledge. You are expected to:

- Master all topics listed in the curriculum.
- Apply Playwright basics in a project (client or personal) and showcase your work.
- Answer questions to validate your understanding.

Every 15 days, evaluation assignments will be provided to assess progress in Playwright proficiency.
